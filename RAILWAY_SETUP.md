# 🚂 Настройка PostgreSQL на Railway

## Пошаговая инструкция

### 1. Добавление PostgreSQL сервиса

1. Откройте ваш проект на [railway.app](https://railway.app)
2. Нажмите **"Add Service"** (или кнопку "+")
3. Выберите **"Database"** → **"PostgreSQL"**
4. Railway создаст новый PostgreSQL сервис

### 2. Подключение базы данных к приложению

1. Перейдите в настройки **вашего основного сервиса** (не PostgreSQL)
2. Откройте вкладку **"Variables"**
3. Нажмите **"Add Variable"**
4. Добавьте переменную:
   - **Имя**: `DATABASE_URL`
   - **Значение**: `${{ Postgres.DATABASE_URL }}`

### 3. Добавление секретного ключа

Добавьте еще одну переменную:
- **Имя**: `SESSION_SECRET`
- **Значение**: любая длинная случайная строка (например: `my-super-secret-key-12345`)

### 4. Перезапуск сервиса

1. Сохраните переменные
2. Railway автоматически перезапустит ваш сервис
3. Проверьте логи - должно появиться сообщение о подключении к PostgreSQL

## ❌ Если что-то не работает

### Проблема: "База данных недоступна"

**Решение:**
1. Убедитесь что PostgreSQL сервис запущен (зеленый статус)
2. Проверьте что переменная `DATABASE_URL` добавлена в основной сервис
3. Значение должно быть точно: `${{ Postgres.DATABASE_URL }}`
4. Перезапустите сервис

### Проблема: "Переменные PostgreSQL не найдены"

**Решение:**
1. Переменная должна быть в **основном сервисе**, не в PostgreSQL сервисе
2. Имя переменной: `DATABASE_URL` (без пробелов)
3. Значение: `${{ Postgres.DATABASE_URL }}` (с двойными фигурными скобками)

### Проблема: "Соединение отклонено"

**Решение:**
1. Подождите 1-2 минуты после создания PostgreSQL сервиса
2. PostgreSQL сервис должен быть в статусе "Active"
3. Попробуйте перезапустить основной сервис

## 🎯 Альтернативные способы

### Способ 1: Через Railway CLI
```bash
railway add postgresql
railway variables set DATABASE_URL=${{ Postgres.DATABASE_URL }}
```

### Способ 2: Если имя сервиса отличается
Если ваш PostgreSQL сервис называется по-другому, замените `Postgres` на реальное имя:
```
${{ YourPostgresServiceName.DATABASE_URL }}
```

### Способ 3: Прямое подключение
Можно скопировать полный URL из PostgreSQL сервиса:
```
postgresql://postgres:password@host:port/database
```

## 🔒 Безопасность

- Никогда не коммитьте `DATABASE_URL` в код
- Используйте переменные окружения Railway
- `SESSION_SECRET` должен быть уникальным для каждого проекта

## 📞 Поддержка

Если проблемы остаются:
1. Проверьте логи приложения в Railway Dashboard
2. Убедитесь что оба сервиса (приложение и PostgreSQL) запущены
3. Попробуйте пересоздать переменную `DATABASE_URL`
