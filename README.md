# 🎥 Веб-приложение для видеозвонков с PostgreSQL

Простое и удобное веб-приложение для видеозвонков с системой регистрации и авторизации.

## 🚀 Возможности

- **Регистрация и авторизация** - безопасная система входа с PostgreSQL
- **Видеозвонки в реальном времени** - WebRTC технология для качественной связи
- **Создание комнат** - генерация уникальных ID для приватных звонков  
- **Ссылки-приглашения** - отправка прямых ссылок для присоединения к звонку
- **Управление медиа** - включение/выключение камеры и микрофона
- **Демонстрация экрана** - возможность показать свой экран другим участникам
- **Адаптивный дизайн** - работает на всех устройствах
- **Сессии в базе данных** - надежное хранение сессий в PostgreSQL

## 🛠 Технологии

- **Backend**: Node.js, Express, Socket.IO
- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **База данных**: PostgreSQL
- **Аутентификация**: bcrypt, express-session
- **WebRTC**: Для видео/аудио связи
- **Развертывание**: Railway

## 📦 Установка и настройка

### Переменные окружения

Создайте файл `.env` на основе `.env.example`:

```bash
# База данных PostgreSQL (Railway предоставляет автоматически)
DATABASE_URL=postgresql://username:password@host:port/database

# Секретный ключ для сессий
SESSION_SECRET=your-super-secret-session-key-change-this

# Окружение
NODE_ENV=production
```

### Локальная разработка

1. Установите PostgreSQL локально
2. Создайте базу данных и выполните скрипт `database.sql`
3. Установите зависимости:
```bash
npm install
```

4. Запустите сервер:
```bash
npm start
```

### Развертывание на Railway

1. **Создайте PostgreSQL базу на Railway**:
   - В панели Railway добавьте PostgreSQL сервис
   - Railway автоматически создаст `DATABASE_URL`

2. **Подключите репозиторий**:
   - Railway автоматически определит настройки
   - Установит зависимости и запустит приложение

3. **Настройте переменные окружения**:
   - `SESSION_SECRET` - уникальный секретный ключ
   - `NODE_ENV=production`

## 🗄️ Структура базы данных

### Таблица `users`
- `id` - уникальный идентификатор
- `username` - имя пользователя (уникальное)
- `email` - email (уникальный)
- `password_hash` - хешированный пароль
- `created_at` - дата создания
- `updated_at` - дата обновления

### Таблица `session`
- Хранение сессий пользователей
- Автоматическая очистка истекших сессий

### Таблица `rooms`
- История созданных комнат
- Связь с пользователями-создателями

## 🎯 Использование

1. **Регистрация**: 
   - Перейдите на `/register`
   - Заполните форму регистрации
   - Автоматический вход после регистрации

2. **Вход**: 
   - Перейдите на `/login`
   - Введите email и пароль

3. **Создание звонка**: 
   - Нажмите "Создать комнату"
   - Скопируйте ссылку и отправьте участникам
   - Нажмите "Начать звонок"

4. **Присоединение к звонку**:
   - Введите ID комнаты или перейдите по ссылке
   - Разрешите доступ к камере и микрофону

## 🔒 Безопасность

- **Хеширование паролей** с bcrypt (10 раундов)
- **Защищенные сессии** в PostgreSQL
- **Валидация данных** на клиенте и сервере
- **HTTPS обязателен** для WebRTC (Railway предоставляет автоматически)
- **Уникальные ограничения** на email и username
- **Автоматическая очистка** истекших сессий

## 🐛 Устранение проблем

### Ошибки подключения к базе данных
- Проверьте `DATABASE_URL` в переменных окружения
- Убедитесь что PostgreSQL сервис запущен на Railway
- Проверьте SSL настройки для продакшена

### Проблемы с сессиями
- Убедитесь что таблица `session` создана
- Проверьте `SESSION_SECRET` в переменных окружения
- Очистите куки браузера при проблемах

### WebRTC проблемы
- Убедитесь что сайт работает через HTTPS
- Проверьте разрешения браузера на камеру/микрофон
- Проверьте настройки файрвола для STUN серверов
