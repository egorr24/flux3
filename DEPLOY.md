# 🚀 Развертывание на Railway

## Пошаговая инструкция

### 1. Подготовка проекта
Убедитесь, что все файлы созданы:
- ✅ `package.json` - зависимости Node.js
- ✅ `server.js` - основной сервер
- ✅ `railway.json` - конфигурация Railway
- ✅ `Procfile` - команда запуска
- ✅ `public/` - статические файлы

### 2. Создание проекта на Railway

1. Перейдите на [railway.app](https://railway.app)
2. Войдите через GitHub
3. Нажмите "New Project"
4. Выберите "Deploy from GitHub repo"
5. Выберите ваш репозиторий с кодом

### 3. Автоматическое развертывание

Railway автоматически:
- Определит Node.js проект
- Установит зависимости (`npm install`)
- Запустит сервер (`npm start`)
- Предоставит публичный URL

### 4. Настройка домена (опционально)

1. В панели Railway перейдите в Settings
2. В разделе "Domains" нажмите "Generate Domain"
3. Или добавьте свой кастомный домен

### 5. Переменные окружения

Railway автоматически установит:
- `PORT` - порт для приложения
- Другие переменные не требуются для базовой функциональности

## 🔧 Локальное тестирование

Перед развертыванием протестируйте локально:

```bash
# Установка зависимостей
npm install

# Запуск сервера
npm start
```

Откройте `http://localhost:3000`

## 🌐 После развертывания

1. Откройте URL вашего приложения на Railway
2. Создайте комнату для тестирования
3. Откройте ссылку в другой вкладке/браузере
4. Разрешите доступ к камере/микрофону
5. Проверьте видеосвязь

## 🔒 Важные замечания

- **HTTPS обязателен** - WebRTC работает только через HTTPS (Railway предоставляет автоматически)
- **Разрешения браузера** - пользователи должны разрешить доступ к камере/микрофону
- **Совместимость** - поддерживаются современные браузеры (Chrome 60+, Firefox 60+, Safari 12+)

## 🐛 Устранение проблем

### Ошибка "Unexpected token '<'"
- Проверьте, что статические файлы отдаются правильно
- Убедитесь, что `express.static` настроен корректно

### Нет доступа к камере/микрофону
- Проверьте, что сайт работает через HTTPS
- Убедитесь, что пользователь разрешил доступ
- Проверьте настройки браузера

### Не работает соединение между пользователями
- Проверьте консоль браузера на ошибки
- Убедитесь, что Socket.IO подключается корректно
- Проверьте, что STUN серверы доступны

## 📱 Мобильные устройства

Приложение работает на мобильных устройствах, но:
- Требуется современный браузер
- Может потребоваться разрешение в настройках браузера
- Рекомендуется использовать в полноэкранном режиме
